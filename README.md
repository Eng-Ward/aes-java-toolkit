# aes-java-toolkit
# AES Encryption Toolkit (Java)

A from-scratch, educational implementation of the **AES (Advanced Encryption Standard)** block cipher in Java, supporting **AES-128** and **AES-192**, with **ECB**, **OFB**, and **CFB** modes of operation. Includes an interactive console driver, file I/O helpers, and optional email delivery of ciphertext/key material via SMTP.

> ⚠️ **Educational project.** This implementation is intended for learning how AES works internally (key expansion, S-box substitution, ShiftRows, MixColumns, etc.). It has **not** been audited or hardened for production use — see [Security Notes](#security-notes) before using it for anything real.

## Features

- **Core AES algorithm implemented from first principles**
  - Key expansion (`AESCore.keyExpansion`)
  - ByteSub / InvByteSub (S-box substitution)
  - ShiftRows / InvShiftRows
  - MixColumns / InvMixColumns (Galois field arithmetic)
  - AddRoundKey
- **Key sizes:** AES-128 (10 rounds) and AES-192 (12 rounds)
- **Modes of operation:**
  - ECB (Electronic Codebook) — with PKCS-style padding
  - OFB (Output Feedback)
  - CFB (Cipher Feedback)
- **Interactive CLI** (`AESDriver`) to configure keys/modes, inspect key expansion, and encrypt/decrypt text or files
- **File handling** for reading/writing binary data, hex-encoded key/IV/nonce files, and encrypted (`.aes`) files
- **Email integration** (JavaMail) to send ciphertext or key material to a recipient
- **Utility helpers** for hex conversion, random key/IV/nonce generation, and formatted console output

## Project Structure

```
├── AESConstants.java   # S-box, inverse S-box, Rcon, MixColumns matrices
├── AESCore.java        # Core AES primitives: key expansion, round transformations
├── AESModes.java       # ECB / OFB / CFB mode implementations + padding
├── AESUtils.java        # Hex <-> bytes conversion, random key/IV generation, console formatting
├── FileHandler.java    # Reading/writing binary, text, and hex files
├── EmailSender.java    # SMTP email sending (ciphertext / key material) via JavaMail
└── AESDriver.java      # Interactive command-line menu tying everything together
```

## Requirements

- **Java 17+** (uses switch expressions / arrow syntax)
- **JavaMail (Jakarta Mail / javax.mail) and JAF (javax.activation)** on the classpath — only required if you use the email-sending features (menu options 7 & 8)

## Getting Started

### 1. Compile

If you're not using the email features, you can compile everything except the mail dependency, or simply have the JavaMail/Activation jars available:

```bash
# Example using javac directly (adjust classpath to where your JavaMail jars live)
javac -d out -cp "lib/*" src/aes/*.java
```

### 2. Run

```bash
java -cp "out:lib/*" aes.AESDriver
```

You'll be greeted with an interactive menu:

```
════════════════════════════════════════════════════════════
  AES APPLICATION MENU                Key: (not set)  Mode: ECB
════════════════════════════════════════════════════════════
   0. Display Current Configuration
   1. Select AES Key Size  (128 / 192 bits)
   2. Key Input            (random or manual / file)
   3. Key Expansion        (display round keys W[i])
   4. Select Cipher Mode   (ECB / OFB / CFB)
   5. Encrypt
   6. Decrypt
   7. Send Ciphertext via Email
   8. Send AES Key / IV / Nonce via Email
   9. Exit
════════════════════════════════════════════════════════════
```

### Typical Workflow

1. **Select key size** (option 1) — choose AES-128 or AES-192.
2. **Provide a key** (option 2) — generate randomly, type a hex string, or load from a file.
3. **Choose a cipher mode** (option 4) — ECB, OFB (requires a nonce), or CFB (requires an IV).
4. **Encrypt** (option 5) — input plaintext from the console or a file, output ciphertext to the console or an `.aes` file.
5. **Decrypt** (option 6) — reverse the process using the same key/mode/IV/nonce.
6. *(Optional)* **Inspect key expansion** (option 3) to see the round keys generated by the key schedule.
7. *(Optional)* **Email ciphertext or key material** (options 7 & 8) via SMTP (Gmail App Passwords recommended).

## Usage Example (Programmatic)

You can also use the core classes directly without the CLI:

```java
byte[] key = AESUtils.generateKey(16); // AES-128
AESCore core = new AESCore(key);
AESModes modes = new AESModes(core);

byte[] plaintext = "Hello, AES!".getBytes();
byte[] ciphertext = modes.ecbEncrypt(plaintext);
byte[] decrypted  = modes.ecbDecrypt(ciphertext);

System.out.println(new String(decrypted)); // "Hello, AES!"
```

## Security Notes

This project is meant for **learning and demonstration**, not production cryptography. Before using it for anything sensitive, be aware of the following:

- **ECB mode leaks patterns** in the plaintext and should generally be avoided for real-world data; prefer CFB/OFB (or better, an authenticated mode like GCM, which is not implemented here).
- **No authentication/integrity** is provided (no MAC or AEAD) — ciphertext can be tampered with undetected.
- **IV/Nonce reuse** in OFB/CFB modes breaks confidentiality; always use a fresh, unpredictable IV/nonce per encryption.
- **Email credentials**: `EmailSender` sends plaintext SMTP credentials (or App Passwords) via `Authenticator`. Never hard-code real credentials in source control; use environment variables or a secrets manager.
- **Sending key material by email** (menu option 8) is inherently risky — anyone with mailbox access can read the key. Use only for demos/testing.
- This implementation has not been checked for timing side-channels and should not be considered constant-time.

For production use cases, prefer a vetted library implementation (e.g., `javax.crypto` / `Cipher` with `AES/GCM/NoPadding`, or a well-audited third-party library).
